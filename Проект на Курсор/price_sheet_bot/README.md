# Бот: прайсы из Telegram → min / 2-я цена в F/G с цветом поставщика

## Как пользоваться (основной режим)

### Вариант A — метка перед прайсом (как у вас: «AL»)

1. Отправьте **отдельным коротким сообщением** код поставщика: `AL`, `Celltech`, и т.д.  
2. **Следующим** сообщением пришлите текст прайса.  
3. Все цены из этого прайса учитываются как от **AL** (и цвет в таблице берётся по ключу `AL` в `supplier_colors.json`).

Команда **`/cancel`** сбрасывает ожидаемую метку, если передумали.

### Вариант B — пересланный прайс

1. **Перешлите** боту сообщение с прайсом.  
2. Имя поставщика берётся из **автора пересланного** сообщения (например «Леша icloud»).  
3. Бот разбирает строки вида:
   - `iPhone 17e 256GB @573$` и ниже `Black 45`, `White 45` (цена из `@`);
   - `17 Pro 256GB` и ниже `Orange 60 x 1175$`;
   - `White (4 x 775$)` и т.п.  
4. Для **каждой позиции** (модель + цвет) в памяти накапливаются цены **разных** поставщиков (файл `data/offers.json`).  
5. В таблицу пишутся **минимальная** и **вторая** цена по рынку, с **цветом** из `supplier_colors.json` по **имени поставщика из Telegram**.

Добавьте в **`supplier_colors.json`** ключ с **точным или близким** именем, как в Telegram (пример уже есть: `Леша icloud`). Цвета — `red`, `green`, `blue` от 0 до 1.

## Таблица

- **Колонка A** — наименование (строки с `DATA_START_ROW`);  
- **F** — первая цена; **G** — вторая;  
- Совпадение с A — по **нечёткому** названию (как раньше).

**Страны в прайсе (флаги):** если в сообщении есть **строка из нескольких одних и тех же флагов** (🇯🇵 или 🇬🇧) без цен и без названия модели, дальше все позиции относятся к этой секции: **🇯🇵** → к ключу товара добавляется **`eSIM`**, **🇬🇧** → **`SIM`** (значения по умолчанию, см. `REGION_SUFFIX_JP` / `REGION_SUFFIX_GB` в `.env`). В колонке A должны быть **отдельные строки** для Японии и UK, например `… Orange eSIM` и `… Orange SIM`, иначе бот не найдёт ячейку.

## Ручной режим

Если переслать нельзя:  
1-я строка — **как в колонке A**; далее строки `Celltech 638`, `Otm 640` (без разбора типичного Telegram-прайса).

## Настройка

`.env` — см. `.env.example`. Важно: `SPREADSHEET_ID`, `WORKSHEET_NAME`, `credentials.json`, при необходимости `DATA_START_ROW`, `FIRST_PRICE_COLUMN`, `SECOND_PRICE_COLUMN`.
Накопление предложений: `OFFERS_STORE_PATH` (по умолчанию `data/offers.json`).

**Доступ Google Sheets:** e-mail сервисного аккаунта из `credentials.json` (поле `client_email`, вид `…@….iam.gserviceaccount.com`) нужно **добавить в таблицу** с правом **Редактор**. Без этого запись в лист даст ошибку (часто HTTP 403). Текст ошибки от API теперь дублируется в ответе бота; полный traceback смотрите в терминале, где запущен `python bot.py`.

**Имя листа (`WORKSHEET_NAME`):** должно совпадать с **вкладкой внизу** таблицы (не с названием бота в Telegram). Лишние пробелы в конце имени вкладки в Google **игнорируются** при поиске. Если имени нет в списке, можно задать `WORKSHEET_FALLBACK_NAMES` или оставить `WORKSHEET_FALLBACK_TO_FIRST=1`, чтобы писать в первый лист.

```bash
pip install -r requirements.txt
python bot.py
```

## Перезапуск бота

После любых правок в коде нужно **остановить** процесс и **запустить снова**, иначе Telegram продолжит работать со старой версией.

1. В терминале, где запущен бот, нажмите **Ctrl+C** (macOS/Linux) или закройте окно терминала.
2. Перейдите в каталог проекта и снова запустите:

```bash
cd "/Users/pavelisaenko/Documents/New project/price_sheet_bot"
source venv/bin/activate   # если используете виртуальное окружение venv
python bot.py
```

Если `venv` не используете — строку с `source` пропустите. Окно терминала с `python bot.py` должно оставаться открытым, пока бот нужен онлайн.
